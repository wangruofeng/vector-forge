# Vector Forge 产品迭代计划

> 最近更新：2026-07-19 ｜ 当前版本：v0.1.0

## 产品定位

**一句话**：面向开发者/设计师的轻量在线 SVG 编辑器——拿到现成 SVG，快速改色、改字、调尺寸、导出。

**非目标**：
- 不做全功能设计工具（不与 Figma / Boxy SVG 正面竞争）
- 不做移动端深度编辑（移动端仅低成本兜底）
- 不引入账号体系（分享走"链接即权限"）

**工程约束**（沿用 AGENTS.md 红线）：
- 零运行时依赖（React/Vite 之外），分享压缩、ID 生成等均手写实现
- 模块化单页架构：`src/main.jsx` 负责编排，SVG 逻辑 / 文案 / 面板 / 状态 Hook 分别下沉到 `src/editor/`、`src/app/`、`src/components/`、`src/hooks/`，界面文案走 `COPY` 双语
- 一切 SVG 变更经 `commitDocument`，保证撤销/重做不断链

---

## 阶段总览

| 阶段 | 主题 | 目标 | 关键交付 |
|---|---|---|---|
| **P1** | 工作流闭环 | 「编辑 → 导出 → 分享」链路完整可用 | PNG/WebP 导出、SVG 瘦身、对齐/分布、右键菜单、多选高亮、分享链接 v1、历史栈上限 |
| **P2** | 专业编辑感 | 具备精细排版能力 | 旋转手柄、网格/标尺/吸附、画布尺寸编辑、模板库扩充、欢迎体验、移动端兜底 |
| **P3** | 增长 | 获客与传播 | 静态 landing 页、分享 v2 短链（Pages Functions + KV）、og 卡片、数据埋点 |
| **P4** | 工程化 | 可持续迭代 | vitest 纯函数测试、CI、大文件性能、错误边界 |

> 原则：P1 全部为零后端、零依赖即可交付的项；需要后端的能力统一收敛到 P3。

## P1：工作流闭环（预计 1–2 周）

| # | 功能 | 说明 | 验收标准 |
|---|---|---|---|
| 1.1 | PNG / WebP 导出 | `<canvas>` + `drawImage` 渲染，零依赖；支持 1x/2x/3x 倍图、透明/白/自定义背景 | 导出面板可选格式与倍率；透明背景导出结果无棋盘格残留 |
| 1.2 | SVG 瘦身与导出清理 | 导出时剥离 `data-editor-id` 等编辑器属性；可选压缩（去空白、小数位收敛） | 导出文件无 `data-editor-id`；压缩选项可减少 30%+ 体积（视文件而定） |
| 1.3 | 对齐 / 分布 | 多选时画布工具栏出现对齐组：左/水平居中/右、顶/垂直居中/底、水平/垂直等间距；基于各元素 bbox 计算，走 `commitDocument` | 多选 ≥2 个元素可对齐；≥3 个可等间距分布；单步撤销可还原 |
| 1.4 | 右键菜单 + 图层重命名 | 图层行与画布元素 `contextmenu`：重命名/复制/粘贴/删除/显隐；重命名写 `data-name` 属性 | 菜单项均可撤销；重命名同步到图层面板与状态栏 |
| 1.5 | 多选画布高亮 | 替换当前单选字符串替换注入（main.jsx `renderedMarkup.replace`），序列化时对全部 `selectedIds` 注入 `is-selected` | 多选时所有选中元素在画布有高亮，不只主选中项 |
| 1.6 | 分享链接 v1（URL hash） | SVG → deflate 压缩 → base64url → `location.hash`；打开带 hash 链接优先于 localStorage | 复制链接新标签页打开可还原文档；hash 缺失/损坏时回退本地文档并提示 |
| 1.7 | 历史栈上限 + 存储降级 | `history.past` 截断至 50 步；localStorage 写入失败时 toast 提示而非静默 | 连续编辑 60+ 步后撤销不丢最近 50 步；存储满时用户可见提示 |

## P2：专业编辑感（预计 2–3 周）

| # | 功能 | 说明 | 验收标准 |
|---|---|---|---|
| 2.1 | 旋转手柄 | 选框上方旋转柄，拖动写 `transform: rotate()`，Shift 吸附 15° | 旋转可撤销；与既有位移/缩放 transform 正确叠加 |

| 2.3 | 画布尺寸 / viewBox 编辑 | 修改 `viewBox` 与画幅，可选内容等比缩放 | 修改后预览/导出一致；不产生内容裁切意外 |
| 2.4 | 模板库扩充 | 内置 6–8 个模板：图标、Logo、Banner、UI 卡片等 | 模板面板可选；加载模板即生成新文档 |
| 2.5 | 欢迎体验 | 首次访问引导层：拖入 SVG / 从模板开始 / 打开示例；`localStorage` 标记不再显示 | 新访客首屏可见三入口；老用户不受影响 |
| 2.6 | 移动端低成本兜底 | 小屏显示「建议桌面端使用」提示，保留只读预览与缩放 | ≤680px 不再出现功能残缺的编辑界面 |

|---|---|---|---|
| 4.1 | vitest 纯函数测试 | 覆盖 `parseSvg`、`updateElementAttributes`、`translateElements`、`insertClonedLayer`、`formatSvgMarkup`、`syncTextLineLayout` 等纯函数 | 核心函数行覆盖 ≥80%；`npm test` 纳入开发流程 |

## 分享链接：存储技术选型

### 候选对比

| 方案 | 适配度 | 延迟 | 成本 | 结论 |
|---|---|---|---|---|
| **A. URL hash（零后端）** | 文本直接编码进链接 | — | 免费 | ✅ P1 采用：当周可上，零依赖零运维 |
| **B. Pages Functions + KV** | 为「读多写少的小文本」设计；25MB value 上限富余；边缘缓存 | 低（边缘） | 免费额度约 10 万读/天、1 千写/天（以官方为准），个人项目富余 | ✅ P3 采用：短链体验，与现有 Pages 项目同域部署 |
| **C. R2 对象存储** | 为大文件/二进制设计（10GB 免费、零出口费） | 中 | 低 | ⚠️ 不作为分享主存储：几 KB 的 SVG 文本用 KV 延迟更低、API 更简单；仅在 3.3 存 og 预览图时引入 |
| **D. D1 (SQLite)** | 关系库 | — | 低 | ❌ 简单 KV 场景引入 SQL 过重 |

### 结论

1. **P1 用 URL hash**：`svg → deflate → base64url → #d=...`。零成本、数据不上服务器（隐私友好）。已知短板：超长链接可能被 IM 截断——分享 UI 中提示，并由 v2 兜底。
2. **P3 用 KV 短链**：`POST /api/share` 生成不可枚举短 ID（自实现，~10 字符、58 进制 alphabet）写入 KV；`/s/:id` 读取还原。Pages 项目根加 `functions/` 目录即可，与现有 Direct Upload 流程兼容。
3. **R2 定位**：社交预览图与未来素材库（大文件）的存储，不参与 SVG 文本分享主链路。

### 配套策略

- **权限模型**：链接即权限，无账号；ID 保证足够熵（≥64bit）防枚举。
- **大小上限**：KV 写入限 512KB，超限提示用户走「导出文件」或 URL hash。
- **过期策略**：默认永久；如需清理，KV TTL 设 180 天（写入时可配）。
- **降级**：KV 不可用时回退提示「当前仅支持导出文件」。

## 度量指标（P3 埋点后可见）

- 核心漏斗：访问 → 导入/模板创建 → 编辑 ≥1 次 → 导出/分享
- 功能采用率：PNG 导出占比、对齐功能使用频次、模板使用率
- 分享健康度：分享创建数、短链打开数、hash 链接失败回退率
- 质量：导入失败率、存储写入失败率

## 风险与开放问题

| 风险 | 缓解 |
|---|---|
| `main.jsx` / `useCanvasInteractions.js` 持续膨胀（main.jsx 当前 ~970 行） | P4 测试先行兜底；功能以纯函数沉淀到 `src/editor/`，组件继续下沉到 `src/components/`，main.jsx 只保留编排 |
| KV 免费额度超限（分享爆发） | 写入加尺寸/频率限制；监控用量，必要时升级付费（$5/月起） |
| hash 链接被 IM 截断 | 分享面板标注「链接较长」，P3 短链兜底 |
| landing 页与编辑器路由拆分影响现有部署 | 先用 Vite MPA 多页验证，再切 `/` + `/app`；保留旧链接重定向 |

**开放问题**：
- 模板库内容来源：纯手工维护 or 开放投稿？（建议 P2 先手工 6–8 个）
- 分享是否支持「只读视图」（打开者不进入编辑态）？（建议 v2 再做，v1 打开即编辑器）
